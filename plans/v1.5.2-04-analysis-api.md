# Feature Implementation Plan: Phase 4 Enhanced Analysis API (v1.5.2)

## 🔍 Analysis & Context
*   **Objective:** Enhance the analysis APIs to include aggregated power curve with HR data, cleaned time-in-zones to ignore power spikes, and introduce a new efficiency trend endpoint tracking aerobic fitness over time.
*   **Affected Files:**
    *   `server/database.py`
    *   `server/ingest.py`
    *   `server/models/schemas.py`
    *   `server/queries.py`
    *   `server/routers/analysis.py`
    *   `tests/test_api.py`
*   **Key Dependencies:** `psycopg2` for advanced window functions and `DISTINCT ON`.
*   **Risks/Edge Cases:**
    *   Ingestion must gracefully handle legacy/corrupted fit files without heart rate data (default to NULL).
    *   Using Postgres-specific window queries requires ensuring `date::date` casting works with existing `TEXT` dates.
    *   Existing frontend may expect `avg_hr` to optionally not exist, which is handled via schema updates.

2.  **Step 2.B (Update compute logic):**
    *   *Target File:* `server/ingest.py`
    *   *Exact Change:* Rename `compute_rolling_best` to `compute_rolling_best_power_and_hr(powers, hrs, window_s)`:
        ```python
        def compute_rolling_best_power_and_hr(powers, hrs, window_s):
            if len(powers) < window_s or len(hrs) < window_s:
                return None, None
            current_power_sum = sum(powers[:window_s])
            current_hr_sum = sum(hrs[:window_s])
            best_power = current_power_sum
            best_hr_sum = current_hr_sum
            
            for i in range(window_s, len(powers)):
                current_power_sum += powers[i] - powers[i - window_s]
                current_hr_sum += hrs[i] - hrs[i - window_s]
                if current_power_sum > best_power:
                    best_power = current_power_sum
                    best_hr_sum = current_hr_sum
                    
            avg_hr = round(best_hr_sum / window_s) if best_hr_sum > 0 else None
            return round(best_power / window_s), avg_hr
        ```
3.  **Step 2.C (Extract and Calculate):**
    *   *Target File:* `server/ingest.py`
    *   *Exact Change:* In `parse_ride_json`:
        - Extract HR data: `hrs = [r.get("heart_rate", 0) or 0 for r in records]`
        - Update calls for individual scalar bests (e.g. `best_1min, _ = compute_rolling_best_power_and_hr(powers, hrs, 60) if powers else (None, None)`).
        - Update loop for `POWER_BEST_DURATIONS`:
          ```python
          for dur in POWER_BEST_DURATIONS:
              best_p, best_hr = compute_rolling_best_power_and_hr(powers, hrs, dur)
          ```

    *   *Exact Change:* Update `get_power_bests_rows` to eliminate the complex self-join and use Postgres's `DISTINCT ON` to safely fetch exactly one absolute best power per duration with its associated `avg_hr`:
        ```python
        def get_power_bests_rows(conn, start_date: str | None = None, end_date: str | None = None) -> list[dict]:
            if start_date or end_date:
                sd = start_date or "2000-01-01"
                ed = end_date or datetime.now().strftime("%Y-%m-%d")
                rows = conn.execute(
                    """SELECT DISTINCT ON (duration_s) duration_s, power, avg_hr, date, ride_id
                       FROM power_bests
                       WHERE date >= %s AND date <= %s
                       ORDER BY duration_s, power DESC, date DESC""",
                    (sd, ed),
                ).fetchall()
            else:
                rows = conn.execute(
                    """SELECT DISTINCT ON (duration_s) duration_s, power, avg_hr, date, ride_id
                       FROM power_bests
                       ORDER BY duration_s, power DESC, date DESC"""
                ).fetchall()
            return [dict(r) for r in rows]
        ```

#### Phase 5: Efficiency Trend Analysis
1.  **Step 5.A (The Unit Test Harness):**

    *   *Test Cases to Write:* Add `test_efficiency_trends(client)` to ensure `GET /api/analysis/efficiency-trends` returns HTTP 200 and matches the expected JSON structure (`date`, `ride_id`, `ef`, `rolling_ef`, `duration_s`).

    *   *Target File:* `server/routers/analysis.py`
    *   *Exact Change:* Create a new `@router.get("/efficiency-trends")` endpoint using Postgres `INTERVAL` math for the rolling average:
        ```python
        @router.get("/efficiency-trends")
        def efficiency_trends(
            start_date: Optional[str] = Query(None),
            end_date: Optional[str] = Query(None),
            user: CurrentUser = Depends(require_read),
        ):
            """Aerobic fitness trend using EF (Normalized Power / Avg HR) for endurance rides."""
            query = """
                WITH filtered_rides AS (
                    SELECT
                        id as ride_id,
                        date,
                        duration_s,
                        (normalized_power::float / avg_hr) as ef,
                        AVG(normalized_power::float / avg_hr) OVER (
                            ORDER BY date::date
                            RANGE BETWEEN INTERVAL '30 days' PRECEDING AND CURRENT ROW
                        ) as rolling_ef
                    FROM rides
                    WHERE sport = 'cycling'
                      AND duration_s >= 1800
                      AND intensity_factor BETWEEN 0.5 AND 0.8
                      AND avg_hr > 0
                      AND normalized_power > 0
                )
                SELECT * FROM filtered_rides
            """
            params = []
            if start_date:
                query += " WHERE date >= ?"
                params.append(start_date)
            if end_date:
                query += " AND date <= ?" if start_date else " WHERE date <= ?"
                params.append(end_date)
            query += " ORDER BY date ASC"

            with get_db() as conn:
                rows = conn.execute(query, params).fetchall()

            return [
                {
                    "date": r["date"],
                    "ride_id": r["ride_id"],
                    "ef": round(r["ef"], 3),
                    "rolling_ef": round(r["rolling_ef"], 3) if r["rolling_ef"] else None,
                    "duration_s": r["duration_s"]
                }
                for r in rows
            ]
        ```
