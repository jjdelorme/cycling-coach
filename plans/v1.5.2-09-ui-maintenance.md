# Campaign 9: UI Maintenance & Infrastructure (v1.5.2)

**Goal:** Address minor UI regressions and improve development transparency with version indicators.

## Tasks

### 1. Calendar Day Hover/Selection Color Contrast
**Status:** Planned

The core selection/highlight mechanism is working, but the hover and selection state colors are currently the same shade of grey used for days belonging to the previous/following months (e.g., April 1st, 2nd, 3rd shown in March's view). This causes visual confusion.

**Fix:** Update the highlight/hover color used for selected/hovered days to a distinct shade or color so it can be easily distinguished from the "out-of-month" greyed-out days.

**Verify:** Check that hovering over and clicking a day in the current month produces a clear, distinct visual state that does not blend in with adjacent month days.

### 2. Dev Build Version String
Update build scripts to display `version#-dirty-commithash` format during development, making it easy to identify exactly which code is running.

### 3. UI Version Footer
Add a small version indicator in the bottom-right corner of every page showing the current UI version. Should be unobtrusive (small text, muted color).

## Success Criteria
- [ ] Calendar day selection and hover states are distinctly colored from adjacent month days.
- [ ] Dev builds show dirty/commit info in version string.
- [ ] Version footer visible on all pages.
