# Campaign 9: UI Maintenance & Infrastructure (v1.5.2)

**Goal:** Address minor UI regressions and improve development transparency with version indicators.

## Tasks

### 1. Calendar Day Selection Highlight
**Status:** Needs verification — may already be fixed.

The MD3 redesign (commit `425ca91` / `ba19743`) changed the selected day highlight from a visible `ring-2 ring-accent ring-inset` to a subtle `bg-surface-low` background, which makes the selection nearly invisible since `bg-surface-low` (#16162a) is *darker* than the normal surface (#1e1e36).

**Fix (if still needed):** Restore a visible selection indicator — either bring back the accent ring or use `bg-surface-high` with a border to make selected days stand out.

**Verify:** Check that clicking a day in the Calendar view produces a clear visual highlight and that the detail panel appears correctly.

### 2. Dev Build Version String
Update build scripts to display `version#-dirty-commithash` format during development, making it easy to identify exactly which code is running.

### 3. UI Version Footer
Add a small version indicator in the bottom-right corner of every page showing the current UI version. Should be unobtrusive (small text, muted color).

## Success Criteria
- [ ] Calendar day selection is clearly visible.
- [ ] Dev builds show dirty/commit info in version string.
- [ ] Version footer visible on all pages.
