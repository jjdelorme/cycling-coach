# Feature Implementation Plan: Foundations & Metric Integrity (v1.5.2)

## 🔍 Analysis & Context
*   **Objective:** Establish a robust mathematical and data foundation for training metrics (CTL, ATL, TSS, NP) using high-performance libraries, and expand the database schema to support advanced power analytics and data cleaning.
*   **Affected Files:**
    *   `requirements.txt`
    *   `server/database.py`
    *   `server/queries.py`
    *   `server/ingest.py`
    *   `tests/test_metrics.py` (to be created)
*   **Key Dependencies:** `numpy`, `scipy`, `psycopg2`
*   **Risks/Edge Cases:** Refactoring the core ingest pipeline to use NumPy/SciPy must not break existing TSS/CTL/ATL values. Time-series data often has gaps or spikes (dropouts/outliers) which must be gracefully handled by the new signal processing logic. The `athlete_log` history must accurately align with ride dates to prevent retroactive changes to historical TSS if FTP changes.

#### Phase 1: Environment & Tooling
1.  **Step 1.A (Add dependencies):** Update requirements to support vectorized math.
    *   *Target File:* `requirements.txt`
    *   *Exact Change:* Append `numpy>=1.24.0` and `scipy>=1.10.0` to the file.
    *   *Justification:* NumPy/SciPy are essential because standard Python loops over 1-second records (e.g., ~14,400 rows for a 4-hour ride) are computationally slow. A 30s rolling average for NP requires $O(N \times K)$ operations in pure Python but is a highly optimized $O(N)$ vectorized operation using `numpy.convolve`. SciPy provides `scipy.interpolate` for gap filling and outlier detection which is far more robust than manual iteration.

#### Phase 3: Metric Source-of-Truth
1.  **Step 3.A (The Unit Test Harness):** Test historical lookups.
    *   *Target File:* `tests/test_queries.py`
    *   *Test Cases to Write:* Setup `athlete_settings` with FTP=250 and `athlete_log` with FTP=200 on '2023-01-01' and FTP=220 on '2023-06-01'. Assert `get_latest_metric('ftp', '2023-05-01') == 200`. Assert `get_latest_metric('ftp', '2024-01-01') == 250`.
2.  **Step 3.B (The Implementation):** Build `get_latest_metric`.
    *   *Target File:* `server/queries.py`
    *   *Exact Change:* Implement helper that queries `athlete_log` for the most recent metric entry prior to a given date, falling back to `athlete_settings`.
        ```python
        def get_latest_metric(conn, key: str, as_of_date: str) -> float:
            # Try athlete_log first for historical accuracy
            row = conn.execute(
                "SELECT value FROM athlete_log WHERE type = %s AND date <= %s ORDER BY date DESC LIMIT 1",
                (key, as_of_date)
            ).fetchone()
            if row and row["value"] is not None:
                return float(row["value"])
            
            # Fallback to current settings
            row = conn.execute(
                "SELECT value FROM athlete_settings WHERE key = %s",
                (key,)
            ).fetchone()
            return float(row["value"]) if row else 0.0
        ```
    *   *Note:* Replace raw FTP/Weight lookups in `server/ingest.py` and `server/routers/pmc.py` with this consistent helper.

#### Phase 4: Signal Processing & Math Vectorization
1.  **Step 4.A (The Unit Test Harness):** Define mathematical accuracy constraints.
    *   *Target File:* `tests/test_metrics.py`
    *   *Test Cases to Write:*
        *   NP: Given an array of exactly 300W for 1 hour, NP=300. Given alternating 100W and 300W blocks, calculate correct 30s rolling average, 4th power mean, and 4th root.
        *   CTL/ATL: Provide predefined TSS sequence and assert EWMA values match established training software outputs exactly.
2.  **Step 4.B (The Implementation):** Implement NP, TSS, CTL, ATL.
    *   *Target File:* `server/ingest.py` (or new `server/metrics.py`)
    *   *Algorithms:*
        *   **CTL:** 42-day EWMA. $CTL_{today} = CTL_{yesterday} + (TSS_{today} - CTL_{yesterday}) / 42$.
        *   **ATL:** 7-day EWMA. $ATL_{today} = ATL_{yesterday} + (TSS_{today} - ATL_{yesterday}) / 7$.
        *   **NP:** 
            ```python
            import numpy as np
            # Pad beginning with zeros or repeating first value depending on standard
            window = np.ones(30) / 30
            # 30-second rolling average
            roll_avg = np.convolve(power_array, window, mode='valid')
            # 4th power, mean, and 4th root
            np_val = np.mean(roll_avg ** 4) ** 0.25
            ```
        *   **TSS:** $TSS = \frac{duration\_s \times NP^2}{FTP^2 \times 36}$
    *   *Data Cleaning (SciPy):*
        *   Missing values (gaps < 10s): `scipy.interpolate.interp1d` (linear interpolation).
        *   Outlier detection: Zero out power > 2500W or sudden spikes using `scipy.signal.medfilt`.
    *   *Exact Change:* Extract the PMC calculation loops from `server/ingest.py` and upgrade them to utilize vectorized NumPy arrays for the full history when recomputing PMC. Apply the SciPy cleaning pipeline to the `ride_records` before deriving `power_bests` and `NP`.
3.  **Step 4.C (The Verification):** Verify PMC reconstruction.
    *   *Action:* Run `pytest tests/test_metrics.py`.
    *   *Success:* All historical metrics generate accurately without performance degradation.

### 🧪 Global Testing Strategy
*   **Unit Tests:** Pure math functions (NP, TSS, hrTSS, CTL, ATL) must be tested with known input arrays and expected outputs to verify floating-point precision and formula correctness.
