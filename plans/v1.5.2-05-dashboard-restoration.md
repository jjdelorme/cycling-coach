# Phase 5: Dashboard & UI Restoration (v1.5.2)

**Goal:** Restore the "Actual vs Planned" visual context to the dashboard by re-implementing the ghost bar overlay that was removed during the MD3 redesign (commit `425ca91`).

## Technical Research Summary

### What Was Lost
The ghost bar visualization was originally added in commit `696468a` and removed as a regression in `425ca91` ("Phase 2: Refine Dashboard with MD3 Tonal Surfaces"). The backend data pipeline is fully intact — only the frontend chart rendering was lost.

### Current State
- **Backend**: No changes needed. The `planned_workouts` table has `planned_tss` and `total_duration_s`. The `fetchWeekPlan()` API already returns this data.
- **Frontend**: `Dashboard.tsx` already fetches `plannedWeeks` via `fetchWeekPlan()` for the "Next Workout" card, but no longer uses it for the weekly volume chart. The `planMondays` computation and `plannedWeeks` query are already in place.
- **API**: `GET /api/rides/summary/weekly` returns only actuals. Adding `planned_tss`/`planned_duration_h` to the response is **not needed** — the old implementation fetched planned data separately via `GET /api/plan/week/{date}`, which still works. This avoids any breaking API changes.

### Consumers of `/api/rides/summary/weekly`
Only one active consumer: `Dashboard.tsx` (via `useWeeklySummary()`). Also invalidated by `useDeleteRide()`. The `WeeklySummary` schema is safe to extend but does not need changes for this campaign.

## Implementation Plan

### 1. Restore Ghost Bar Chart (Frontend Only)

Re-implement the weekly volume chart in `Dashboard.tsx` using the pattern from commit `696468a`, adapted to the current MD3 styling.

#### 1a. Restore Helper Functions
The following already exist in `Dashboard.tsx` but were removed from the chart:
- `planMondays` — already computed (lines 45-57)
- `plannedWeeks` — already fetched (lines 60-63)

**Add back:**
- `isoWeekToMonday(isoWeek: string): string` — converts `"2026-W14"` to `"2026-03-30"` for matching actuals to plans.
- `thisMonday` — add to the existing `planMondays` useMemo (was previously included, then removed).
- `plannedByMonday` useMemo — aggregates `plannedWeeks` into a `Map<string, { tss: number; hours: number }>` keyed by Monday date.

#### 1b. Replace Simplified Chart with Ghost Bar Version
Replace the current simple bar chart (two datasets: TSS + Hours) with the full version:

- **Unified week list**: Last 12 actual weeks + up to 3 future planned weeks (only if they have planned data).
- **Color logic**:
  - Past/current weeks with actuals: solid bars (`rgba(59, 130, 246, 0.7)` for TSS, `rgba(34, 197, 94, 0.7)` for Hours).
  - Future planned-only weeks: faded bars (`rgba(..., 0.25)`).
- **Ghost bar overlay**: A custom Chart.js plugin (`plannedOverlayPlugin`) that draws dashed, semi-transparent rectangles behind the current week's actual bars showing planned targets.
  - Uses `beforeDatasetsDraw` hook to render behind actual bars.
  - Draws on both y-axes (TSS left, Hours right).
  - Dashed border (`setLineDash([4, 4])`) with low-alpha fill.
- **Current week highlight**: Bold blue x-axis label for the current week via scriptable `ticks.color` and `ticks.font`.
- **Enhanced tooltips**: Show "TSS: 280 / 350 planned" format for the current week when planned data exists.

#### 1c. Ghost Bar UX Rules
- Ghost bars appear **only** for weeks that have planned workouts. No plan = no ghost bar.
- Future weeks with plans but no rides show only the faded planned bars.
- Past weeks show only solid actual bars (no ghost bars for historical comparison).
- Current week shows both: solid actual bar with dashed ghost bar overlay behind it.

### 2. Sport Name Mapping

#### 2a. Sport Values in the Database
Sports come from two sources:
- **JSON/FIT imports**: Stored lowercase (e.g., `cycling`, `virtualride`, `mountainbikeride`).
- **Intervals.icu sync**: Uses their taxonomy (e.g., `Ride`, `VirtualRide`, `MountainBikeRide`, `Run`, `Hike`).

The full taxonomy is documented in `AGENTS.md` under "Intervals.icu Activity Taxonomy."

#### 2b. Add `fmtSport()` to `frontend/src/lib/format.ts`
