# Plan: Restructure Settings Page and Restrict Access

## Objective
- Restructure the Settings page into a tabbed interface (Athlete, Coach, System).
- Restrict sensitive settings (API keys, AI config, Sync) into a "System" tab visible only to `admin` users.
- Group display and physiological settings into an "Athlete" tab.
- Group AI coaching configurations into a "Coach" tab.
- Allow `read` only users to see "Athlete" and "Coach" settings (view-only), but completely hide the "System" tab and "Admin" (Users) page.

## Key Files & Context
- `frontend/src/components/Layout.tsx`: Navigation guards.
- `frontend/src/App.tsx`: Route/Component guards.
- `frontend/src/pages/Settings.tsx`: Tabbed UI implementation and role-based filtering.

## Implementation Steps

### 1. Update Layout Navigation Guards
- In `frontend/src/components/Layout.tsx`:
    - The "Settings" button remains visible to `read`, `readwrite`, and `admin` roles.
    - The "Users" (admin) button remains restricted strictly to the `admin` role.

### 2. Update App Route Guards
- In `frontend/src/App.tsx`:
    - Ensure the `{tab === 'admin' && <Admin />}` block is guarded with `user?.role === 'admin'`.
    - No specific guard needed for `{tab === 'settings' && <Settings />}` as all authenticated users (including `read`) should see it.

### 3. Restructure Settings Page
- In `frontend/src/pages/Settings.tsx`:
    - Add a `type Tab = 'athlete' | 'coach' | 'system'` state.
    - Implement a tab navigation header (styled similarly to the Analysis page).
    - Group sections:
        - **Athlete Tab:** (Visible to `read`, `readwrite`, `admin`)
            - **Display Section:** Units (Imperial/Metric) and Theme (Dark/Light).
            - **Athlete Profile Section:** Physiological metrics (LTHR, Max HR, FTP, Weight, etc.).
        - **Coach Tab:** (Visible to `read`, `readwrite`, `admin`)
            - **Coach Configuration Section:** AI coach instructions (Athlete Profile, Principles, Coach Role, Plan Management).
        - **System Tab:** (Visible ONLY to `admin`)
            - **Gemini AI Section:** Model, Location, and API Key settings.
            - **Integrations Section:** intervals.icu API key/ID and the "Sync Now" functionality (including progress/logs).
            - **Maintenance:** "Reset to Defaults" button and Version Info.
    - **Permissions Logic:**
        - `read` only users: Can see **Athlete** and **Coach** tabs. "Save" buttons in these tabs will be hidden or disabled. They cannot see the **System** tab.
        - `readwrite` users: Can see **Athlete** and **Coach** tabs. Can use "Save" buttons in these tabs. They cannot see the **System** tab.
        - `admin` users: Can see and edit all tabs (**Athlete**, **Coach**, and **System**).

## Verification & Testing
- **Role Verification:**
    - Log in as **Admin**: Verify all tabs and Users page are accessible and editable.
    - Log in as **ReadWrite**: Verify Athlete and Coach tabs are accessible and editable. Verify System tab and Users page are hidden.
    - Log in as **Read**: Verify Athlete and Coach tabs are accessible (view-only). Verify System tab and Users page are hidden.
- **Functionality Verification:**
    - Test saving settings in Athlete and Coach tabs.
    - Test "Sync Now" functionality in the System tab for Admin users.
    - Ensure "Theme" toggle still works correctly.
