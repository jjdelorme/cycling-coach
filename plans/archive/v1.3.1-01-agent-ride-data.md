# Plan: Agent Ride Data Access + Best Efforts Analysis

**Version:** 1.3.1
**Type:** Feature
**Complexity:** Medium
**Estimated Lines:** ~400-450

## Summary

Give the AI coaching agent access to per-second ride data and best-effort analysis via four new tools using a progressive drill-down pattern. Best efforts are computed server-side as part of the ride analysis tool.

## Current State

- The agent has 7 read tools in `server/coaching/tools.py`, none exposing per-second record data
- `ride_records` table stores one row per second per ride (power, HR, cadence, speed, altitude, etc.)
- `compute_rolling_best()` already exists in `server/ingest.py` for sliding-window power bests (returns best avg only, no start index)
- `power_bests` table stores pre-computed bests at standard durations (5s, 30s, 1min, 5min, 20min, 60min)
- REST endpoint `GET /api/analysis/power-curve` exists but has no agent tool equivalent
- Zone distribution exists in `server/routers/analysis.py` (`zone_distribution()`) -- reuse the same zone boundaries
- Athlete settings store `ftp`, `lthr`, `max_hr`, `resting_hr` -- use for zone calculations

## Files to Modify

- `server/coaching/tools.py` — add four new tool functions plus helpers
- `server/coaching/agent.py` — import and register new tools
- `tests/test_coaching_tools.py` — add unit tests

### Algorithm: Enhanced `compute_rolling_best_with_index`

```python
def _compute_rolling_best_with_index(values: list[int | None], window_s: int) -> tuple[int, int] | None:
    """Sliding window best average with start index tracking.
    Returns (best_avg, start_index) or None if len(values) < window_s.

    Algorithm:
        1. Replace None with 0 in values
        2. Compute initial sum of first window_s values
        3. Track best_sum = initial_sum, best_start = 0
        4. Slide window: for i in range(window_s, len(values)):
             current_sum += values[i] - values[i - window_s]
             if current_sum > best_sum:
                 best_sum = current_sum
                 best_start = i - window_s + 1
        5. Return (round(best_sum / window_s), best_start)
    """
```

### Algorithm: Zone Distribution

```python
def _compute_power_zones(powers: list[int | None], ftp: int) -> dict:
    """Zone boundaries (matching server/routers/analysis.py):
        Z1: < 55% FTP, Z2: 55-75%, Z3: 75-90%, Z4: 90-105%, Z5: 105-120%, Z6: >= 120%
        Coasting: power == 0 or None
    """
```

### Algorithm: HR Drift

```python
def _compute_hr_drift(powers: list, heart_rates: list) -> dict | None:
    """Filter to samples where both power > 0 and HR > 0.
    Split into first/second half. Compare avg HR.
    Return None if fewer than 600 valid samples.
    drift_pct = 100 * (second_half_avg_hr - first_half_avg_hr) / first_half_avg_hr
    """
```

### Algorithm: Fatigue Decoupling

```python
def _compute_decoupling(powers: list, heart_rates: list) -> dict | None:
    """Pw:Hr ratio first vs second half.
    NP via 30s rolling avg, 4th power. decoupling_pct = 100 * (first_np_hr - second_np_hr) / first_np_hr.
    Positive = decoupling = aerobic system fading. Return None if < 1200 valid samples.
    """
```

### SQL for fetching ride records

```sql
SELECT power, heart_rate, cadence, speed, altitude
FROM ride_records WHERE ride_id = ? ORDER BY id
```

### SQL for all-time bests

```sql
SELECT duration_s, MAX(power) as max_power FROM power_bests GROUP BY duration_s
```

### Error Handling

- No ride found: return `{"error": "No ride found for date {date}"}`
- No power data: set `has_power = False`, skip `best_efforts` and `power_zones`, add warning
- No HR data: set `hr_zones = None`, `hr_drift = None`, `decoupling = None`
- Short ride (< 300s): add warning
- FTP == 0: skip power zones, add warning
- Best efforts: omit durations longer than the ride

## Phase 3: Tool 2 — `get_ride_segments(date, segment_duration_s=300)`

### Function Signature

```python
def get_ride_segments(date: str, segment_duration_s: int = 300) -> dict:
    """Get a ride's data as averaged time segments for trend analysis.
    Downsamples per-second data into segments (default 5-minute windows).
    Args:
        date: Date string (YYYY-MM-DD).
        segment_duration_s: Window size in seconds (min 60, max 1800).
    """
```

### Return Data Structure

```python
{
    "ride_id": int, "date": str, "segment_duration_s": int, "segment_count": int,
    "segments": [
        {
            "segment": int,             # 1-indexed
            "start_elapsed_s": int,
            "avg_power": int | None, "max_power": int | None, "min_power": int | None,
            "avg_hr": int | None, "max_hr": int | None,
            "avg_cadence": int | None,
            "avg_speed_kph": float | None,
            "altitude_gain_m": float | None,
            "distance_m": float | None,
        },
    ],
}
```

### Algorithm

1. Resolve date -> ride_id. Clamp segment_duration_s to [60, 1800].
2. Fetch all records. Group into chunks of segment_duration_s.
3. For each chunk: compute mean/max/min of non-None values, altitude gain from positive deltas.
4. Final segment may be shorter — still include it.

## Implementation Notes

1. Zone boundaries must match `server/routers/analysis.py` `zone_distribution()`.
2. Read FTP from ride's own `ftp` field first, fall back to `get_athlete_setting("ftp")`.
3. Performance: `cursor.fetchall()` for single-ride records is fine (~10K rows max).
4. NP in decoupling: 30-second rolling average, 4th root of mean of 4th powers (Coggan formula).
5. Place helper functions (underscore-prefixed) at top of `tools.py` after imports.
6. Add `from server.database import get_db, get_athlete_setting` at top of `tools.py`.
