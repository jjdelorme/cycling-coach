# Feature Implementation Plan: Unified Sync Recalculation

## 🔍 Analysis & Context
*   **Objective:** Unify the metric calculation pipeline by extracting the core logic from `server/ingest.py` into a reusable, **stateless** function. This function will be invoked during the Intervals.icu sync in `server/services/sync.py` to ensure synced rides receive proper metrics and power bests. The design will prioritize scalability for multi-athlete support by offloading heavy CPU-bound tasks.
*   **Affected Files:** `server/ingest.py`, `server/services/sync.py`, `server/metrics.py`, `tests/test_sync.py`, `tests/test_ingestion.py`.
*   **Key Dependencies:** `scipy` (for data cleaning), PostgreSQL (for updating rides and power_bests).
*   **Risks/Edge Cases:** 
    * Intervals.icu streams might have missing arrays (e.g., HR but no power), requiring the processor to gracefully degrade.
    * **Scalability:** Large stream arrays could block the event loop. We must utilize `asyncio.to_thread` for the metrics processing call to ensure the main thread remains responsive for multiple concurrent users.

## 📝 Step-by-Step Implementation Details

### Prerequisites
None.

#### Phase 1: Preparation & Test Harness
1.  **Step 1.A (The Unit Test Harness):** Define verification for metric extraction.
    *   *Target File:* `tests/test_metrics.py` or `tests/test_ingestion.py`
    *   *Test Cases to Write:* Create a unit test for `process_ride_samples` using dummy lists of `raw_powers`, `raw_hrs`, `raw_cadences`. Assert that it correctly delegates to `clean_ride_data`, `calculate_np`, `calculate_tss`, and `compute_rolling_best`, returning a dictionary of structured metrics (NP, IF, TSS, vi, power bests). Ensure the function remains stateless (no database calls within the processor).
2.  **Step 1.B (Integration Test Harness):** Define verification for sync.
    *   *Target File:* `tests/test_sync.py`
    *   *Test Cases to Write:* Mock `fetch_activity_streams` to return valid power/time data. Assert that after calling `_download_rides`, there are entries in the `power_bests` table for the synced ride.

#### Phase 2: Refactoring Core Logic
1.  **Step 2.A (The Extraction):** Create the reusable, stateless pipeline function.
    *   *Target File:* `server/metrics.py` (Recommended for pure logic) or keep in `server/ingest.py`.
    *   **Exact Change:** Create a function `process_ride_samples(raw_powers: list, raw_hrs: list, raw_cadences: list, ftp: float, duration_s: float, lthr: float = 0, max_hr: float = 0, resting_hr: float = 0) -> dict`. Move the logic currently in `parse_ride_json` spanning from `# 3. Clean using SciPy pipeline` down to `# Power bests at standard durations`. 
            *   Include logic to compute `hrTSS` using `compute_hr_tss` if `has_power_data` is false but `raw_hrs` are present.
            *   **Design Rule:** The function must remain stateless (input data in -> dictionary out).
            *   Return a dictionary containing `np_power`, `tss`, `avg_power`, `intensity_factor`, `variability_index`, `has_power_data`, `data_status`, and a list of `power_bests` dictionaries.
    2.  **Step 2.B (Update Ingestion):** Make `parse_ride_json` use the new function.
        *   *Target File:* `server/ingest.py`
        *   *Exact Change:* 
            *   Fetch `lthr`, `max_hr`, and `resting_hr` (via existing `get_latest_metric` calls) before calling `process_ride_samples`.
            *   Replace the inline calculation logic in `parse_ride_json` with a call to `process_ride_samples`. Use the returned dictionary to populate the `ride` dictionary and the `power_bests` list.

3.  **Step 2.C (The Verification):** Verify ingestion works as before.
    *   *Action:* Run `pytest tests/test_ingestion.py`.
    *   *Success:* Tests pass without regression.

#### Phase 3: Integration into Sync Pipeline (Scalable)
1.  **Step 3.A (The Implementation):** Calculate metrics during sync without blocking.
    *   *Target File:* `server/services/sync.py`
    *   *Exact Change:* Inside `_download_rides`, immediately after `streams = await asyncio.to_thread(fetch_activity_streams, icu_id)` and `_store_streams`, extract `raw_powers = streams.get('watts', [])`, `raw_hrs`, and `raw_cadences`. 
    *   **Implementation:** Call `process_ride_samples` inside `asyncio.to_thread(process_ride_samples, ...)` to offload the CPU-heavy calculations from the event loop.
2.  **Step 3.B (Persistence):** Save the calculated values.
    *   *Target File:* `server/services/sync.py`
    *   *Exact Change:* 
        *   Execute an `UPDATE rides SET normalized_power=?, tss=?, intensity_factor=?, variability_index=?, best_1min_power=... WHERE id=?` using the results.
        *   Iterate over the returned `power_bests` array and insert them into the `power_bests` table using a bulk `executemany` statement.
3.  **Step 3.C (The Verification):** Verify sync writes power bests.
    *   *Action:* Run `pytest tests/test_sync.py`.
    *   *Success:* The mocked sync test from Phase 1 now passes, validating the DB inserts.

### 🧪 Global Testing Strategy
*   **Unit Tests:** Verify `process_ride_samples` robustly handles missing inputs (e.g., all `None` arrays) without throwing exceptions, safely returning 0s and empty lists.
*   **Integration Tests:** End-to-end sync mock ensuring `power_bests` table grows relative to the downloaded sync streams and the event loop remains unblocked.
