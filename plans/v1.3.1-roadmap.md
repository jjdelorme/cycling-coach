# v1.3.1 Point Release Roadmap

**Branch:** `release/1.3.1`
**Created:** 2026-03-28

## Sequence

Work should proceed in this order. Items 1-2 are foundational infrastructure that other items benefit from. Items 3-7 are independent and can be parallelized.

| # | Plan | Type | Est. Effort |
|---|------|------|-------------|
| 1 | [Server Logging](v1.3.1-06-server-logging.md) | Infrastructure | ~6h |
| 2 | [Sync Performance](v1.3.1-04-sync-performance.md) | Performance | ~6h |
| 3 | [Duplicate Workouts](v1.3.1-05-duplicate-workouts.md) | Bug | ~6h |
| 4 | [Interval Segments Import](v1.3.1-02-interval-segments.md) | Data Gap | ~4h |
| 5 | [Agent Ride Data + Best Efforts](v1.3.1-01-agent-ride-data.md) | Feature | ~5h |
| 6 | [Ride Highlight Averages](v1.3.1-03-ride-highlight-averages.md) | Regression | ~3h |
| 7 | [API Token for Cron Sync](v1.3.1-07-api-token-cron.md) | Feature | ~1h |

## Rationale for Ordering

1. **Logging first** — Phase 1 (~1h) turns on all existing silent log calls. Every subsequent item benefits from visibility. Sync timing instrumentation (Phase 2-3) directly feeds into diagnosing sync performance.

2. **Sync performance second** — `executemany` batch fix is a single-line change with 10-50x impact on insert-heavy operations. Interval segment import (#4) and agent data tools (#5) both insert/query `ride_records`, so they benefit from the perf fix being in place.

3. **Duplicate workouts** — Independent bug fix. Can start as soon as logging is in place (helps diagnose the issue further if needed).

4. **Interval segments** — Adds `ride_laps` table and backfills from JSON. Foundation data that the agent ride analysis tools (#5) can optionally leverage.

5. **Agent ride data + best efforts** — The largest feature. Benefits from interval segments being available (can reference laps in analysis). Benefits from sync perf (faster record queries).

6. **Ride highlight averages** — Pure frontend, fully independent. Can be done in parallel with anything.

7. **API token** — Smallest item (~1h). Simple CLI addition. Can be done anytime but logically comes last since it's most useful once sync is fast and reliable.

## Deferred

- [ADK Tools vs REST API Refactor](backlog.md) — Out of scope. Requires architectural rethinking beyond a point release.
