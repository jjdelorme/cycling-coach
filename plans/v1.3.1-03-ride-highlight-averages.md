# Plan: Ride Highlight Averages (Selection Stats)

**Version:** 1.3.1
**Type:** Regression
**Complexity:** Medium
**Estimated Lines:** ~120-150 net new

## Summary

Port the click-drag selection feature that shows averages for a highlighted portion of a ride chart. The old vanilla JS implementation (commits a1a61e6, 36766d7) used direct canvas event listeners, a crosshair plugin, and DOM innerHTML for stats. The React port follows the existing `stepHighlightPlugin` pattern.

## Root Cause

The feature was never ported from `web/js/charts.js` to the React `RideTimelineChart` component (commit aff9821 — React rewrite). Only workout step highlighting (table hover) was implemented.

## 1. Selection Plugin Data Structure (WeakMap)

Follow the exact pattern of `highlightDataMap` (line 595). Create at module scope:

```typescript
const selectionDataMap = new WeakMap<ChartJS, {
  state: 'idle' | 'dragging' | 'locked'
  startIdx: number | null
  endIdx: number | null
}>()
```

## 2. State Machine

| Current State | Event | Next State | Action |
|---|---|---|---|
| `idle` | mousedown inside chartArea | `dragging` | Set startIdx/endIdx to clicked data index |
| `dragging` | mousemove | `dragging` | Update endIdx, `chart.draw()` |
| `dragging` | mouseup, `abs(endIdx-startIdx) > 2` | `locked` | Freeze, compute stats, update React state |
| `dragging` | mouseup, `abs(endIdx-startIdx) <= 2` | `idle` | Clear (too small, treat as click) |
| `dragging` | mouseleave | `idle` | Clear |
| `locked` | mousedown in chartArea | `idle` | Clear selection and React stats |

## 3. Pixel-to-Index Conversion

```typescript
const rect = canvas.getBoundingClientRect()
const mouseX = e.clientX - rect.left
const xScale = chart.scales.x
const rawIdx = xScale.getValueForPixel(mouseX)
const idx = Math.round(Math.max(0, Math.min(rawIdx, dataLength - 1)))
```

Reverse: `xScale.getPixelForValue(idx)` — same as `stepHighlightPlugin` (lines 625-626).

`dataLength` = downsampled array length (chart labels), NOT `records.length`.

## 4. Selection Rectangle Rendering (Plugin `afterDraw`)

```typescript
const selectionPlugin = {
  id: 'selectionHighlight',
  afterDraw(chart: ChartJS) {
    const sel = selectionDataMap.get(chart)
    if (!sel || sel.startIdx == null || sel.endIdx == null || sel.state === 'idle') return
    const { ctx, chartArea, scales } = chart
    if (!chartArea || !scales?.x) return
    const x1 = scales.x.getPixelForValue(Math.min(sel.startIdx, sel.endIdx))
    const x2 = scales.x.getPixelForValue(Math.max(sel.startIdx, sel.endIdx))
    ctx.save()
    ctx.fillStyle = 'rgba(0, 212, 170, 0.12)'
    ctx.fillRect(x1, chartArea.top, x2 - x1, chartArea.bottom - chartArea.top)
    ctx.strokeStyle = 'rgba(0, 212, 170, 0.5)'
    ctx.lineWidth = 1
    ctx.strokeRect(x1, chartArea.top, x2 - x1, chartArea.bottom - chartArea.top)
    ctx.restore()
  },
}
ChartJS.register(selectionPlugin)
```

Uses `afterDraw` (on top of data). `stepHighlightPlugin` uses `beforeDraw` (dims background). Both coexist.

## 5. Stats Computation

When transitioning to `locked`, compute from **original `records` array** (not downsampled):

```typescript
const downsampleStep = Math.max(1, Math.floor(records.length / 600))
const origStart = lo * downsampleStep
const origEnd = Math.min(hi * downsampleStep, records.length - 1)
const slice = records.slice(origStart, origEnd + 1)

const durationSec = slice.length
const powers = slice.map(r => r.power).filter(v => v != null && v > 0)
const hrs = slice.map(r => r.heart_rate).filter(v => v != null && v > 0)
const cadences = slice.map(r => r.cadence).filter(v => v != null && v > 0)
const avg = (arr) => arr.length ? Math.round(arr.reduce((a, b) => a + b, 0) / arr.length) : null
```

`useEffect` for canvas event listeners:
- Initialize WeakMap entry for chart
- Set `canvas.style.cursor = 'crosshair'`
- Attach mousedown/mousemove/mouseup/mouseleave handlers
- Cleanup: remove listeners, reset cursor, delete WeakMap entry
- Dependencies: `[records, workout]`

## 8. Edge Cases

1. **Minimum selection**: Require `abs(endIdx - startIdx) > 2` (3+ points)
2. **Chart boundaries**: Clamp indices with `Math.max(0, Math.min(idx, dataLength - 1))`
3. **stepHighlightPlugin coexistence**: No conflict — different lifecycle hooks and visual layers
4. **Downsample mapping**: Multiply chart index by step, cap at `records.length - 1`
5. **Missing values**: Filter out null/0 before averaging
6. **Unmount during drag**: `useEffect` cleanup removes all listeners
7. **Chart re-render while locked**: WeakMap state persists across `chart.update('none')`
