# Feature Implementation Plan: Ingestion Engine (v1.5.2-02)

## 🔍 Analysis & Context
*   **Objective:** Implement a robust, NumPy-vectorized "Coaching Math" layer to handle stochastic data, sensor errors, and extract high-fidelity metabolic metrics (Power/HR/Cadence coupling) from ride streams.
*   **Affected Files:**
    *   `server/services/power_cleaner.py` (New)
    *   `server/services/efficiency_cleaner.py` (New)
    *   `server/ingest.py`
    *   `scripts/repair_data.py` (New)
*   **Key Dependencies:** `numpy`, `scipy.interpolate.interp1d`
*   **Risks/Edge Cases:**
    *   Missing sensor data (e.g., HR strap drops for 5 mins) causing gaps that shouldn't be blindly interpolated.
    *   Performance bottlenecks if looping over 10,000+ points per ride in pure Python.
    *   Database schema constraints (e.g., `rides.data_status` needs to exist or be added, and `power_bests` might need new columns for HR/Cadence).

2.  **Step 1.B (The Implementation - PowerCleaner):** Execute the core change.
    *   *Target File:* `server/services/power_cleaner.py`
    *   *Exact Change:* Create a class/module with the exact statistical methods:
        *   Convert the power array to a NumPy array.
        *   Calculate a 60s rolling mean and standard deviation using `np.convolve` or Pandas rolling equivalents (or a custom NumPy stride trick).
        *   Calculate the rolling Z-score: `(power - rolling_mean) / rolling_std`.
        *   Flag indices where `|Z-score| > 3.5`.
        *   Use `scipy.interpolate.interp1d` with `kind='linear'` (and `bounds_error=False`, `fill_value="extrapolate"`) to interpolate the flagged indices using valid surrounding data.

3.  **Step 1.C (The Implementation - EfficiencyCleaner):** Execute the core change.
    *   *Target File:* `server/services/efficiency_cleaner.py`
    *   *Exact Change:* Implement physiological bounds logic to flag sensor dropouts or "static electricity" spikes.
        *   Convert HR array to NumPy array.
        *   Compute rate-of-change (derivative): `np.abs(np.diff(hr, prepend=hr[0]))`.
        *   Flag indices as invalid if:
            *   `HR < 40` or `HR > 220` (Absolute physiological bounds).
            *   Rate-of-change `> 20` bpm/sec.
        *   Return a boolean mask of valid indices (or `np.nan` for invalid points) to be used when calculating Efficiency Factor (EF) or Avg HR for efforts.

2.  **Step 2.B (The Implementation - Rolling Best):** Execute the core change.
    *   *Target File:* `server/ingest.py`
    *   *Exact Change:* Rewrite `compute_rolling_best(powers, hr, cadence, window_s)` using NumPy vectorization.
        *   Use `np.convolve(powers, np.ones(window_s), 'valid') / window_s` to compute all rolling averages efficiently.
        *   Find the `start_offset_s` (the `argmax` of the convolved array).
        *   Extract the slice `[start_offset_s : start_offset_s + window_s]` for HR and Cadence arrays.
        *   Return a dictionary: `{"power": best_power, "start_offset_s": start_offset_s, "avg_hr": mean_hr, "avg_cadence": mean_cad}`.
