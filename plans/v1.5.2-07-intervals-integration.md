# Feature Implementation Plan: Intervals.icu Integration Enhancements (v1.5.2-07)

## 🔍 Analysis & Context
*   **Objective:** Enhance the bi-directional communication with Intervals.icu by implementing write-back capabilities for athlete metrics (FTP/Weight) and resolving issues with duplicate workout creation.
*   **Affected Files:**
    - `server/services/intervals_icu.py` (API helpers)
    - `server/routers/athlete.py` (Integration with Settings UI)
    - `server/coaching/planning_tools.py` (Integration with AI Coach tool)
    - `server/services/sync.py` (Duplicate prevention logic)
*   **Key Dependencies:** Intervals.icu API.
*   **Risks/Edge Cases:**
    - **API Limits:** Frequent updates to wellness data.
    - **Race Conditions:** Simultaneous sync and planning updates causing duplicate entries.

## 📝 Step-by-Step Implementation Details

### 1. Athlete Metric Updates
Implement the following endpoints in `server/services/intervals_icu.py`:

#### 1.1 Updating FTP
- **Endpoint**: `PUT /api/v1/athlete/{athleteId}/sport-settings/Ride`
- **Payload**: `{"ftp": value}`
- **Note**: Use `0` for `{athleteId}`.

#### 1.2 Updating Weight
- **Endpoint**: `PUT /api/v1/athlete/{athleteId}/wellness/{date}`
- **Payload**: `{"weight": value}`
- **Date Format**: `YYYY-MM-DD`.

#### 1.3 Triggering Sync
- Modify `server/routers/athlete.py` and `server/coaching/planning_tools.py` to call these `intervals_icu` functions whenever `ftp` or `weight_kg` keys are updated.
- Ensure the AI Coach tool description is updated to inform the agent that these changes are synchronized externally.

### 2. Duplicate Workout Prevention
- Implement a check to verify if a workout with the same date/name/content already exists on Intervals.icu before pushing a new one.
- Utilize the `icu_id` mapping if available for planned workouts.

## 🎯 Success Criteria
- [x] FTP and Weight can be updated on Intervals.icu via the app UI.
- [x] The AI Coach agent's `update_athlete_setting` tool successfully pushes updates to Intervals.icu.
- [x] Workouts are no longer duplicated on Intervals.icu during planning sync.
